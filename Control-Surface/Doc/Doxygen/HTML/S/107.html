  11 /**
  12  * @brief   A class that groups MIDI_Element%s and allows the user to change
  13  *          the MIDI addresses (Controller number or Note number) or the MIDI
  14  *          channels of these elements.
  15  *
  16  * Consider two controls: `volume_A` and `volume_B`:
  17  * `volume_A` has controller 0x07 on MIDI channel 1, and
  18  * `volume_B` has controller 0x07 on MIDI channel 2.
  19  * Let's add them to a bank with 2 tracks per bank.
  20  * @code{.cpp}
  21  * AnalogCC volume_A(A0, MIDI_CC::Channel_Volume, 1);
  22  * AnalogCC volume_B(A1, MIDI_CC::Channel_Volume, 2);
  23  *
  24  * Bank bank(2);
  25  *
  26  * bank.add(volume_A, Bank::CHANGE_CHANNEL);
  27  * bank.add(volume_B, Bank::CHANGE_CHANNEL);
  28  * @endcode
  29  * By default, the bank setting is 0, so it has no impact:
  30  * `volume_A` still sends MIDI messages on channel 1, and
  31  * `volume_B` still sends MIDI messages on channel 2.
  32  * However, when the bank setting is changed, this is
  33  * no longer the case:
  34  * @code{.cpp}
  35  * bank.setBankSetting(1);
  36  * @endcode
  37  * Now that the bank setting is set to one, the number of
  38  * tracks per bank is added to the channel of each element
  39  * in the bank:
  40  * `volume_A` now sends MIDI messages on channel 3, and
  41  * `volume_B` now sends MIDI messages on channel 4.
  42  *
  43  * When the bank setting is set to 2, `volume_A` will use
  44  * channel 5, and `volume_B` channel 6, etc.
  45  *
  46  * When using the bank type Bank::CHANGE_ADDRESS, the same
  47  * thing happens, but the address changes and the channel
  48  * stays the same.
  49  *
  50  * To change the bank setting, a BankSelector can be used.
  51  *
  52  * Apart from changing the address or channel of an element,
  53  * banks can also be used to apply a single map function
  54  * to many Potentiometer (CCPotentiometer or PBPotentiometer)
  55  * elements at the same time.
  56  *
  57  * ### In general
  58  * `Bank::CHANGE_CHANNEL`
  59  * @f$ c = c_0 + t · s @f$ &emsp; where @f$c@f$ is the effective channel,
  60  * @f$c_0@f$ is the base channel of the MIDI_Element, @f$t@f$ is the number of
  61  * tracks per bank, and @f$s@f$ is the current bank setting.
  62  *
  63  * `Bank::CHANGE_ADDRESS`
  64  * @f$ a = a_0 + t · s @f$ &emsp; where @f$a@f$ is the effective address,
  65  * @f$a_0@f$ is the base address of the MIDI_Element, @f$t@f$ is the number of
  66  * tracks per bank, and @f$s@f$ is the current bank setting.
  67  */
