   9 /**
  10  * @brief   A class for extended IO elements.
  11  *          
  12  *          The limited number of IO pins of the Arduino can be extended by
  13  *          adding multiplexers, shift registers, IO expanders, etc.  
  14  *          ExtendedIOElement serves as a base class for all these expanders.  
  15  *          
  16  *          The pins of each extended IO element are mapped to a pin number
  17  *          greater than the greatest Arduino pin number.  
  18  *          You can supply this pin number to the IO functions in the ExtIO
  19  *          namespace.  
  20  *          If the pin number corresponds to an actual Arduino pin,
  21  *          the default Arduino IO function (digitalRead, digitalWrite, ...)
  22  *          will be called.  
  23  *          If the pin is not an Arduino pin, it is an extended IO pin number,
  24  *          so the extended IO element that this pin belongs to is looked up,
  25  *          and the IO function for this element is executed with the correct
  26  *          pin number.
  27  * 
  28  *          For example:  
  29  *          Imagine an Arduino with 20 pins (e.g. the Arduino UNO).
  30  *          Pins 0 - 19 will correspond to the Arduino pins, and
  31  *          `ExtIO::digitalRead(pin)` will have the exact same effect as
  32  *          the standard `digitalRead(pin)` function.  
  33  *          Now, we'll add two 8-channel analog multiplexers, let's call them
  34  *          `mux1` and `mux2`.
  35  *          The first pin (pin 0) of `mux1` will be extended IO pin number 20,
  36  *          the last pin (pin 7) of `mux1` will be extended IO pin number 27,
  37  *          etc.  
  38  *          The first pin of `mux2` will be extended IO pin number 28, you get
  39  *          the idea.  
  40  *          If you now call `ExtIO::digitalRead(mux1.#pin (7))` or 
  41  *          `ExtIO::digitalRead(27)`, both will be
  42  *          translated to `mux1.digitalRead(7)`.
  43  * 
  44  *          The number of extended IO elements is limited only by the size of 
  45  *          #pin_t. However, looking up the extended IO element for a given
  46  *          extended IO pin number uses linear search, so that might add
  47  *          some noticable overhead for large pin numbers.
  48  */
