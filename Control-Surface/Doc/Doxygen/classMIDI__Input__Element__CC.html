◆ updateImpl()

virtual bool MIDI_Input_Element::updateImpl ( const MIDI_message_matcher &  midimsg)
protectedpure virtualinherited

This function is executed when an incoming MIDI message matches this element, and it should update the internal state of this element with the new data of the MIDI message.

Parameters
midimsgThe new incoming MIDI message to update the internal state of the element with.
Returns
true The MIDI message was indeed meant for this element, and the state has been updated.
false Even though the channel and address of the MIDI match this element, the additional addressing information in the data bytes of the MIDI message does not match this element, and the internal state is not updated.

Implemented in MCU_VU, MCU_7SegmentDisplay< LENGTH >, MCU_7SegmentDisplay< AssignmentDisplayLength >, MCU_7SegmentDisplay< TimeDisplayLength >, MCU_VPot_Ring, MCU_7SegmentDisplay< LENGTH >, MCU_7SegmentDisplay< AssignmentDisplayLength >, MCU_7SegmentDisplay< TimeDisplayLength >, and MIDI_Input_Note_Buffer.

◆ updateWith() [2/2]

bool MIDI_Input_Element::updateWith ( const MIDI_message_matcher &  midimsg)
protectedvirtualinherited

Update this MIDI_Input_Element with an incoming MIDI message, if the given message matches this element's channel and address.

If the channel and address of the message matches this element, MIDI_Input_Element::updateImpl is executed, and the internal state is updated with the new data from the MIDI message.

The element moves down in the linked list of MIDI_Input_Elements, so objects that match regularly, will be found faster. (Matching an incoming MIDI message with the elements is basically linear search.)

Parameters
midimsgThe new incoming MIDI message to match against this element.
Returns
true The MIDI message matches this element and the internal state is updated.
false The MIDI message does not match this element. Keep on looking for an element that does match it.
